# Рабочий процесс с Biblio после загрузки на GitHub

## Ежедневная работа с проектом

### 1. Добавление нового материала

Когда у вас есть новый материал для добавления:

1. **Подготовте запрос** для AI-ассистента
2. **Прикрепите файл readme4ai.md** к запросу
3. **Отправьте запрос** - AI определит:
   - Тип контента (промпт, код, сайт, статья)
   - Соответствующую категорию
   - Правильный шаблон для использования
   - Имя файла и метаданные

4. **Создайте файл** сгенерированный AI
5. **Добавьте в Git**:
   ```bash
   git add .
   git commit -m "Add: [название материала]"
   git push origin master
   ```

### 2. Обновление существующего материала

1. **Найдите файл** в соответствующей категории
2. **Откройте его** и внесите изменения
3. **Обновите метаданные** (дата, версия)
4. **Сохраните и отправьте**:
   ```bash
   git add .
   git commit -m "Update: [название материала]"
   git push origin master
   ```

### 3. Работа с AI-ассистентом

**Оптимальный промпт для работы:**
```
Проанализируй этот материал и добавь в библиотеку Biblio. Используй readme4ai.md как инструкцию.

[здесь ваш материал - промпт, описание сайта, код, статья]

Создай файл с правильной структурой, метаданными и на обоих языках.
```

## Структура файлов после загрузки

```
Biblio/                          # Ваш локальный репозиторий
├── README.md                     # Главная страница (не удалять!)
├── categories/                    # Категории контента
│   ├── ai-tools/                # AI-инструменты
│   ├── development/              # Разработка
│   ├── design/                   # Дизайн
│   ├── useful-resources/          # Полезные ресурсы
│   └── knowledge-base/            # База знаний
├── templates/                     # Шаблоны (не удалять!)
│   ├── prompt-template.md
│   ├── tool-template.md
│   ├── website-template.md
│   └── article-template.md
├── tags/                         # Система тегов
├── assets/                        # Изображения и медиа
└── readme4ai.md                  # Инструкция для AI (важна!)
```

## Частые операции

### Добавление изображений

1. **Создайте изображение** в папке `assets/images/`
2. **Используйте относительные пути** в Markdown:
   ```markdown
   ![Описание](../../assets/images/image-name.png)
   ```
3. **Добавьте изображения** в Git:
   ```bash
   git add assets/images/
   git commit -m "Add: images for [material]"
   git push origin master
   ```

### Создание новых категорий

1. **Создайте папку** в `categories/`
2. **Добавьте README.md** и README_EN.md
3. **Обновите главный README.md** с новой категорией
4. **Отправьте изменения**:
   ```bash
   git add .
   git commit -m "Add: new category [name]"
   git push origin master
   ```

### Работа с тегами

1. **Используйте существующие теги** когда возможно
2. **Создавайте новые теги** только при необходимости
3. **Обновляйте tags/index.md** при добавлении новых тегов
4. **Создавайте страницы** в `tags/tag-pages/` для популярных тегов

## Автоматизация с AI

### Шаблон запроса для добавления материала

```
Добавь в Biblio:

**Тип**: [промпт/сайт/код/статья]
**Название**: [название материала]
**Описание**: [краткое описание]
**Категория**: [ai-tools/development/design/useful-resources/knowledge-base]
**Теги**: [тег1, тег2, ru, en]

Используй соответствующий шаблон из templates/ и создай файл с правильной структурой.
```

### Примеры запросов

**Для промпта:**
```
Создай промпт для ChatGPT для написания технической документации. 
Категория: ai-tools/chatgpt-prompts/
Теги: prompt, chatgpt, documentation, technical, ru, en
```

**Для сайта:**
```
Добавь в Biblio сайт Figma - инструмент для дизайна интерфейсов.
Категория: useful-resources/websites/
Теги: website, design, tool, ui, ux, ru, en
```

**Для кода:**
```
Создай сниппет для Python валидации email адресов.
Категория: development/code-snippets/
Теги: code, python, validation, email, utility, ru, en
```

## Проверка качества

Перед отправкой проверьте:

1. ✅ **Двуязычность** - контент на русском и английском
2. ✅ **Метаданные** - все обязательные поля заполнены
3. ✅ **Имя файла** - соответствует формату [тип]-[название].md
4. ✅ **Ссылки** - все ссылки работают корректно
5. ✅ **Изображения** - добавлены в assets/ и правильно подключены

## Решение проблем

### Конфликт при push
```bash
git pull origin master
# Разрешите конфликты
git add .
git commit -m "Resolve conflicts"
git push origin master
```

### Удаление файла
```bash
git delete [имя файла]
git commit -m "Remove: [имя файла]"
git push origin master
```
